프론트엔드 개발에 있어, 가장 중요한 점은 아마 트렌드에 얼마나 빠르게 적응할 수 있느냐라고 생각한다.<br/>
여태 정말 많은 프론트엔드 프레임워크나 라이브러리를 경험해왔지만, 요즘 구인광고를 보면 `React`의 시장 점유율이 굉장히 높은 것으로 보인다.<br/>
`Vue.js` 또한 그에 못지않게 사용되는 것으로 보이는데 다양한 기술 스택이 존재하지만 오늘은 이 두 가지에 대한 설명을 하려고 한다.

## SINGLE PAGE APPLICATION (SPA)

`React`나 `Vue.js`를 알아보기 전에 우선 Single Page Application (SPA) 관하여 알아보아야 한다.<br/>
`SPA` 는 웹 어플리케이션 혹은 웹 사이트로서, 사용자와 상호작용을 할 때 웹 페이지 전체를 다시 로드하는 대신에 필요한 부분만 동적으로 업데이트를 한다.<br/>
기존의 전통적인 웹 어플리케이션에서는 새로운 페이지에 대한 요청이 들어오면 서버에서 전체 페이지를 다시 로드해주는 방식이었지만,
`SPA` 에서는 최초 한 번 페이지 전체를 로드하고 이후 데이터의 변경이 있는 부분만 추가하여 사용자에게 빠르고 부드러운 경험을 제공한다.

- SPA의 특징?
  - **속도** : 필요한 리소스(예를 들어, html, css, javascript)는 단 한 번만 로드되며, 사용자와 상호작용 할 때에는 주로 데이터만 주고받게 되므로 반응성이 빠르다.
  - **동적인 상호작용** : `AJAX`를 활용하여 서버와 비동기적으로 데이터를 주고 받을 수 있다.
  - **사용자 경험** : 페이지 전환 없이도 어플리케이션과 상호작용을 해서, 전체적으로 부드러운 사용자 경험을 제공한다.
  - **서버 부하 감소** : 서버는 주로 데이터를 처리하고, 전체 페이지의 렌더링에 관련한 부분은 담당하지 않기 때문에 서버에 대한 부하가 감소한다.

지금까지의 내용으로만 보면 `SPA`는 사용자 경험 측면에서 아주 우수한 성능을 가지고 있다고 할 수 있다.<br/>
하지만, 장점이 있으면 단점도 있기 마련

- SPA의 단점
  - 초기 로드 시에 모든 리소스를 다운로드해야 하므로 초기 로딩 시간이 다소 길 수 있다.
  - 검색 엔진 최적화 (SEO) 관련한 문제가 생길 수 있다.
    - SPA는 처음에 기본적인 뼈대만을 로딩하고 나머지 콘텐츠는 Javascript를 통해 동적으로 로딩하므로, 대부분의 검색 엔진 크롤러는 웹 페이지의 초기 상태만을 분석하기 때문에,
      인덱싱이 되지 않을 수가 있기 때문
  - 브라우저의 히스토리 관리에 대한 복잡성이 존재한다. 이는 라이브러리나 프레이워크를 사용하여 관리가 가능하다.

> 🚀 &nbsp; SPA에서 사용되는 기술 및 도구로 JavaScript 프레임워크인 `React`, `Vue.js`, `Angular` 등이 있다.

## React

`React`는 Facebook에서 만든 사용자 인터페이스를 구축하기 위한 Javascript 라이브러리이다.

> 🚀 &nbsp; `React`는 전체적인 틀을 모두 제공하는 것이 아닌 하나의 기능만을 활용해서 쓸 수 있는 도구를 제공하기 때문에, 프레임워크가 아닌 라이브러리라고 할 수 있다.

가상 DOM을 사용하여 빠른 렌더링을 가능하게 하고, 컴포넌트 기반의 아키텍쳐를 통해 재사용이 가능한 UI block을 만들 수 있다.<br/>

#### React의 장점

- 컴포넌트 기반 아키텍쳐는 코드 재사용성을 높이고, 대규모 어플리케이션에서도 유지/보수 측면에서 관리가 용이하다.
- 위에 언급했듯이 가상 DOM을 사용하기 때문에 렌더링 속도가 빠르다.
- 많은 사람들이 사용하는 라이브러리이다 보니 커뮤니티의 규모가 크고 풍부한 생태계를 지니고있다.
- 웹 뿐만 아니라 모바일 등 다양한 플랫폼을 지원한다.
- 단방향으로 이루어지는 데이트 바인딩으로 안정성이 굉장히 높다.

## Vue.js

`Vue.js`는 Evan You라는 소프트웨어 개발자에 의해 개발된 프로그레시브 Javascript 프레임워크이다.<br/>
반응형 데이터 바인딩과 컴포넌트 지향적인 개발을 통해서 사용자 인터페이스를 구축하기 위하여 사용된다.

#### Vue.js의 장점

- API 문서가 간결하고 이해하기가 쉬워 러닝커브가 비교적 낮은 편이다.
- `React`와 마찬가지로 가상 DOM을 사용하기 때문에 랜더링 속도가 빠르다.
- 컴포넌트 기반의 아키텍쳐
- 확장성이 높고, 확장 가능한 핵심 라이브러리가 존재한다.

## React.js와 Vue.js의 차이점

- `React`는 UI 라이브러리로, 주로 뷰 레이어에 중점을 두지만, `Vue`는 프레임워크로 어플리케이션 구축의 다양한 측면을 제공한다.
- `React`는 단방향 데이터 흐름을 가지지만, `Vue`는 양방향 데이터 바인딩을 제공한다.
- `React`는 초기 설정이 다소 복잡할 수 있지만, `Vue`는 초기 설정이 더 간단한 반면, `React`에 비해 다양한 라이브러리와 통합이 다소 쉽지 않다.
- `React`는 모든 컴포넌트에서 `JSX`를 사용해 `Vue`의 템플릿에 비해 보다 더 안정적이고 진보된 기능을 사용할 수 있지만 러닝커브가 `Vue`에 비해 높을 수 있음
- `Vue`는 Single File Components를 제공하여 스크립트, 스타일, 마크업이 한 파일 내에서 작성되고, `React`는 `JSX`를 통해 Javascript 내에서 마크업을 작성한다.

## 그래서 결론은?

필자는 `Vue`와 `React`를 모두 경험해보았고 그들의 장단점에 대해서 어느정도 연구하고 느끼고 있다.<br/>
개인의 취향 차이라고도 할 수 있지만, 우선 facebook의 지원이 가능한 `React`가 보다 더 안정적이라고 느껴 `React`를 조금 더 선호하는 편이다.<br/>
또한, `React Native`의 도입으로 `React`의 인기가 더욱 확산되었는데, 이는 웹 뿐만 아니라 모바일 환경에서도 강세를 보이고 있기에 사용성이 큰 `React`를 선호한다.<br/>

_근데 사실 `flutter` 써보니까, `React Native`보다 사용자 경험 측면에서 훨씬 우세한 거 같아서, `flutter` 쓸 줄 안다면 `flutter`가 더 나은 듯._<br/>

물론 사용 용도에 따라, 혹은 클라이언트의 요청에 따라 바뀔 수 있는 부분이기 때문에 어느 쪽이 더 확실히 좋다고 단정지을 순 없다.
